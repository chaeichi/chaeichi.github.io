## 진법
### 10진법과 2진법
- 1946년에 개발된 컴퓨터 `에니악(ENIAC)`
    - 사람에게 익숙한 `10진법`을 사용하도록 설계
    - 전기회로는 전압이 불안정해서 전압을 10단계로 나누어 처리하는데 한계가 있었음
- 1950년에 개발된 `에드박(EDVAC)`
    - 10진법이 아닌 `2진법`을 사용하도록 설계
    - 전기가 흐르면 1, 흐르지 않으면 0만으로 동작하도록 설계
- 그 이후부터 지금까지 대부분의 컴퓨터는 2진 체계로 설계

- 컴퓨터는 2진수(0과 1) 밖에 모르기 때문에 값을 저장하면 2진수로 바뀌어서 저장됨
    - int age = 25; // 변수 age에 25를 저장
    - 10진수 25는 2진수 11001로 바뀌어 저장
    - int타입의 크기가 4byte면 32자리의 2진수로 표현하여 000000000000000000000000000**11001**
- 2진법은 `0`과 `1`로만 데이터를 표현하기 때문에 10진법에 비해 많은 자리수를 필요로 함
- 자리수가 많아지긴 해도 10진수를 온전히 표현할 수 있음
- 덧셈이나 뺄셈 같은 연산도 10진수와 동일함

- n비트로 표현할 수 있는 10진수
    - 값의 개수 : 2<sup>n</sup>
        - 4비트로 10진수를 표현한다면 값의 개수는 2<sup>4</sup> = 16
    - 값의 범위 : 0 ~ 2<sup>n</sup>-1
        - 4비트로 10진수를 표현한다면 값의 범위는 0 ~ 2<sup>4</sup>-1 = 0 ~ 15

### 8진법과 16진법
- `2진법`은 `0`과 `1`, 두 개의 기호만으로 값을 표현하므로 자리수가 상당히 길어짐
- 이러한 단점을 보완하기 위해 `8진법`이나 `16진법` 사용
- `8진수`는 **2진수 3자리**를 한자리로 표현
    - **0~7**을 기호로 사용
- `16진수`는 **2진수 4자리**를 한자리로 표현
    - **0~9**, **A~F**를 기호로 사용
    - 16진수 A는 10진수로 10, F는 10진수로 15

### 정수의 진법 변환
#### 2진수를 8진수, 16진수로 변환
- `8진수` : 뒤에서 3자리씩 끊어서 그에 해당하는 8진수로 바꿈
- `16진수` : 뒤에서 4자리씩 끊어서 그에 해당하는 16진수로 바꿈

#### n진수를 10진수로 변환
- `각 자리의 수`에 `해당 단위의 값`을 곱해서 모두 더하면 된다.

- 101110<sub>(2)</sub><br/>
= 1x2<sup>(5)</sup> + 0x2<sup>(4)</sup> + 1x2<sup>(3)</sup> + 1x2<sup>(2)</sup> + 1x2<sup>(1)</sup> + 0x2<sup>(0)</sup><br/>
= 32 + 8 + 4 + 2<br/>
= 46<sup>(10)</sup>

- 1460<sup>(8)</sup><br/>
= 1x8<sup>(3)</sup> + 4x8<sup>(2)</sup> + 6x8<sup>(1)</sup> + 0x8<sup>(0)</sup><br/>
= 512 + 256 + 48<br/>
= 816<sup>(10)</sup>

- 64F<sub>(16)</sub><br/>
= 6x16<sup>(2)</sup> + 4x16<sup>(1)</sup> + 15x16<sup>(0)</sup><br/>
= 1536 + 64 + 15<br/>
= 1615<sup>(10)</sup>

### 실수의 진법 변환
#### 10진 소수점수를 2진 소수점수로 변환
1. `10진 소수`에 `2`를 곱한다.
    - 0.625 x 2 = 1.25
2. 결과에서 `소수부만` 가져다가 다시 `2`를 곱한다.
    - 0.625 x 2 = 1.**25**
    - **0.25** x 2 = 0.5 
3. 1과 2의 과정을 소수부가 `0`이 될 때까지 반복한다. *(이 때, 소수가 0이 되지 않고 무한히 반복될 수도 있다.)*
    - 0.625 x 2 = 1.25
    - 0.25 x 2 = 0.5
    - 0.5 x 2 = 1.0 
4. 위의 결과에서 `정수부만`을 위에서 아래로 순서대로 적고 `0.`을 앞에 붙이면 된다.
    - 0.625 x 2 = **1**.25
    - 0.25 x 2 = **0**.5
    - 0.5 x 2 = **1**.0
    - 0.625<sub>(10)</sub> = 0.101<sub>(2)</sub>

- 만약 123.456처럼 정수부가 있는 소수점수는 정수부 123과 소수점부 0.456을 따로 변환한 다음에 더하면 된다.

#### 2진 소수점수를 10진 소수점수로 변환
- 0.101<sub>(2)</sub><br/>
= 1x2<sup>-1</sup> + 0x2<sup>-2</sup> + 1x2<sup>-3</sup><br/>
= 1x0.5 + 0x0.25 x 1x0.125<br/>
= 0.5 + 0.125<br/>
= 0.625<sub>(10)</sub><br/>

### 음수의 2진 표현 - 2의 보수법

- `부호없는 정수(0과 양수)`의 표현에 모두 사용하면 **0 ~ 2<sup>n</sup> -1**까지의 정수를 나타낼 수 있음

- `부호있는 정수(양수와 음수)`를 모두 표현하려면?

    - 왼쪽의 `첫 번째 비트(MSB, Most Significant Bit)`가 `0`이면 `양수`, `1`이면 `음수`이므로 첫 번째 비트만으로 값의 부호를 알 수 있게 됨

- 1로 시작하는 2진수를 음수표현에 사용할 경우
    - 양수의 첫 번째 비트만 1로 바꾸면 음수가 된다는 장점 존재
    - 그러나, **0이 두 개(0, -0) 존재**한다는 문제가 발생
    - **두 수를 더했을 때 2진수가 0이 되지 않음**
    - **2진수가 증가할 때 10진 음수는 감소**
    - `2의 보수법`에 의한 음수 배치로 해결 가능

- `2의 보수법`에 의한 음수 배치
    - 절대값이 같은 양수와 음수를 더했을 때 **2진수로도 0의 결과를 얻음**
        - 5<sup>(10)</sup> + -5<sup>(10)</sup><br/>
        = 0101 + 1011<br/>
        = 0<br/>
        (자리올림이 발생해 10000이 되지만 크기가 4비트라 버려짐)
    - **2진수가 증가할 때 10진 음수가 감소한다는 모순이 사라짐**
    - 다만, 첫 번째 비트를 바꾸는 것만으로 값의 부호를 바꿀 수 없게 됨

#### 2의 보수법
- 절대값이 같고 부호가 다른 두 10진수를 표현하는 것
- 어떤 수의 n의 보수는 더했을 때 n이 되는 수
    - 7의 10의 보수는 3
    - 3의 10의 보수는 7
    - 3과 7은 10의 보수의 관계
- 2의 보수 관계 역시 더해서 2가 되는 두 수의 관계를 말함
    - 10진수 2는 2진수로 10
    - 2진수로 10은 **자리올림이 발생하고 0이 되는 수**를 뜻함

#### 음수의 2진 표현을 구하는 방법
1. 음수의 `절대값`을 `2진수`로 변환
    - -5의 절대값인 5를 2진수로 변환하면 0101
2. 1에서 구한 2진수의 `1`을 `0`으로, `0`은 `1`로 바꾼다.(`1의 보수` 구하기)
    - 0101 → 1010
3. 2에서 구한 `1의 보수`에 `1`을 더한다.(`2의 보수` 구하기)
    - 1010 + 1 = 1011

## 비트(bit)와 바이트(byte)
- `비트(bit, binary digit)` : 한 자리의 2진수, 1bit는 `컴퓨터가 값을 저장할 수 있는 최소단위`
- `바이트(byte)` : `8bit`, `데이터의 기본 단위`로 사용
- `워드(word)` : `CPU가 한 번에 처리할 수 있는 데이터의 크기`
    - CPU 성능에 따라 워드에 크기가 달라짐
    - 32bit CPU에서 1word = 32bit(4byte)
    - 64bit CPU에서 1word = 64bit(8byte)
- `니블(nibble)` : `4bit`, 16진수 1자리(2진수 4자리)를 저장할 수 있는 단위
