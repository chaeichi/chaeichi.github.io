## 형변환
### 형변환(캐스팅, casting)이란?
- **변수 또는 상수의 타입을 다른 타입으로 변환하는 것**
- 서로 다른 타입간의 연산을 수행해야하는 경우가 존재
- 두 값을 `같은 타입`으로 변환한 다음에 연산을 수행해야 함
- `기본형(primitive type)`에서 `boolean`을 제외한 나머지 타입들은 **서로 형변환이 가능**

### 형변환 방법
- `(타입)피연산자`
- 형변환하고자 하는 변수나 리터럴 앞에 변환하고자 하는 타입의 괄호와 함께 붙여줌
- `괄호()`는 `캐스트 연산자` 또는 `형변환 연산자`라고 함

```CastingEx1.java
class CastingEx1 {
    public static void main(String[] args) {
        double d = 85.4;
        int score = (int)d;

        System.out.println("score=" + score);
        System.out.println("d=" + d);
    }
}
```

```실행결과
score=85
d=85.4 // 형변환 후에도 피연산자에는 아무런 변화가 없다.
```

### 정수형간의 형변환
- `큰 타입`에서 `작은타입`으로 변환하는 경우 크기의 차이만큼 잘려나가므로 `값 손실(loss of data)` 발생

![표2-21 정수형간의 형변환 - 큰 타입에서 작은 타입으로의 변환](/assets/images/javajungsuk3/2-21-table.png)

- `작은 타입`에서 `큰타입`으로 변환하는 경우 잘려나가는 일이 없으므로 `값 손실이 발생하지 않음`
    - 나머지 빈공간은 `0` 또는 `1`로 채워짐
    - 원래의 값은 채우고 남은 빈공간은 `0`으로 채우는게 보통
    - 변환하려는 값이 `음수`인 경우 `1`로 채움(형변환 후에도 `부호 유지`를 위해서)

![표2-23 정수형간의 형변환 - 큰 타입에서 작은 타입으로의 변환](/assets/images/javajungsuk3/2-23-table.png)

```CastingEx2.java
class CastingEx2 {
    public static void main(String[] args) {
        int i = 10;
        byte b = (byte)i;
        System.out.printf("[int -> byte] i=%d -> b=%d%n", i, b);

        i = 300;
        b = (byte)i;
        System.out.printf("[int -> byte] i=%d -> b=%d%n", i, b);

        b = 10;
        i = (int)b;
        System.out.printf("[byte -> int] b=%d -> i=%d%n", b, i);

        b = -2;
        i = (int)b;
        System.out.printf("[byte -> int] b=%d -> i=%d%n", b, i);

        System.out.println("i=" + Integer.toBinaryString(i));
    }
}
```

```실행결과
[int -> byte] i=10 -> b=10
[int -> byte] i=300 -> b=44
[byte -> int] b=10 -> i=10
[byte -> int] b=-2 -> i=-2
i=11111111111111111111111111111110 // 변수 i의 값인 -2를 2진수로 출력
```

- `Integer.toBinaryString(int i)` : `10진 정수`를 `2진 정수`로 변환한 `문자열`을 얻을 수 있음

### 실수형 간의 형변환
- `float 타입(작은 타입)`의 값을 `double 타입(큰 타입)`으로 변환하는 경우
    - `지수(E)`는 float의 기저인 `127`을 뺀 후 double의 기저인 `1023`을 더해서 변환
    - `가수(M)`는 **23자리를 채우고 남은 자리를 0으로 채움**

- `double 타입(큰 타입)`을 `float 타입(작은 타입)`으로 변환하는 경우
    - `지수(E)`는 double의 기저인 `1023`을 뺀 후 float의 기저인 `127`을 더해서 변환
    - `가수(M)`는 double의 가수 **52자리 중 23자리만 저장되고 나머지는 버려진다.**
    - **가수의 24번째 자리에서 반올림이 발생할 수 있다.**
    - 24번째 자리의 값이 `1`이면 `반올림` 발생
- float 타입의 범위를 넘는 값을 float로 형변환 하는 경우 `±무한대` 또는 `0`을 결과로 얻음

```CastingEx3.java
class CastingEx3 {
    public static void main(String[] args) {
        float f = 9.1234567f;
        double d = 9.1234567;
        double d2 = (double)f;

        System.out.printf("f=%20.18f\n", f);
        System.out.printf("d=%20.18f\n", d);
        System.out.printf("d2=%20.18f\n", d2);
    }
}
```

```실행결과
f=9.123456954956055000
d=9.123456700000000000
d2=9.123456954956055000
```

- float와 double의 정밀도 차이 때문에 서로 다른 값이 저장됨
    - f=9.123456954956055000
    - d=9.123456700000000000
- 저장할 때 이미 값이 달라졌기 때문에 형변환을 해도 값이 같아지지 않음
    - d=9.123456700000000000
    - d2=9.123456954956055000

### 정수형과 실수형 간의 형변환
![그림2-13 int타입과 float타입의 표현형식](/assets/images/javajungsuk3/2-13.png)

- 정수형과 실수형은 저장형식이 다르기 때문에 좀 더 복잡한 변환과정을 거쳐야 함

#### 정수형을 실수형으로 변환
- 정수는 소수점 이하의 값이 없으므로 비교적 변환이 간단
- 정수를 2진수로 변환한 다음 정규화를 거쳐 실수의 저장형식으로 저장
- **실수형의 정밀도의 제한으로 인한 오차가 발생할 수 있음**
    - `int`의 최대값은 약 20억으로 최대 `10자리의 정밀도`를 요구
    - `float`는 10진수로 약 `7자리의 정밀도`만 제공

#### 실수형을 정수형으로 변환
- 실수형을 정수형으로 변환하면 **실수형의 소수점 이하 값은 버려짐**
- **정수형의 표현 형식으로 소수점 이하의 값은 표현할 수 없기 때문**
- 값이 버려지기 때문에 형변환할 때 반올림이 발생하지 않음
- **실수의 소수점을 버리고 남은 정수가 정수형의 저장범위를 넘는 경우에는 정수의 오버플로우가 발생함**

```CastingEx4.java
class CastingEx4 {
    public static void main(String[] args) {
        int i = 91234567; // 8자리의 10진수
        float f = (float)i; // int를 float로 형변환
        int i2 = (int)f; // float를 다시 int로 형변환

        double d = (double)i; // int를 double로 형변환
        int i3 = (int)d; // double을 다시 int로 형변환

        float f2 = 1.666f;
        int i4 = (int)f2;

        System.out.printf("i=%d\n", i);
        System.out.printf("f=%f i2=%d\n", f, i2);
        System.out.printf("d=%f i3=%d\n", d, i3);
        System.out.printf("(int)%f=%d\n", f2, i4);
    }
}
```

```실행결과
i=91234567
f=91234568.000000 i2=91234568
d=91234567.000000 i3=91234567
(int)1.666000=1
```

### 자동 형변환
- 서로 다른 타입간의 대입이나 연산을 할 때, 형변환으로 타입을 일치시키는 것이 원칙
- **형변환을 생략하면 컴파일러가 생략된 형변환을 자동적으로 추가해줌**

```casting
float f = 1234; // 형변환의 생략. 값 손실이 발생하지 않음
byte b = 1000; // 에러 발생. byte의 범위(-128 ~ 127)를 넘는 값을 저장
char ch = (char)1000; // 명시적 형변환. 에러가 발생하지 않음
```

#### 자동 형변환의 규칙
- **기존의 값을 최대한 보존할 수 있는 타입으로 자동 형변환**

![그림2-21 기본형의 자동 형변환이 가능한 방향](/assets/images/javajungsuk3/2-21.png)

- 표현 범위가 `좁은 타입`에서 `넓은 타입`으로 형변환 하는 경우 값 손실이 없음
- 두 타입 중에서 **표현범위가 더 넓은 쪽으로** 형변환 됨
- 반대로 `넓은 타입`에서 `좁은 타입`으로의 형변환은 값 손실이 발생할 수 있으므로 자동 형변환이 되지 않음. `명시적 형변환` 필요
- **실수형은 정수형과 값을 표현하는 방식이 다르기 때문에 같은 크기일지라도 훨씬 더 큰 표현 범위를 가짐**
- 정수형을 실수형으로 형변환 시, 정밀도 한계로 인한 오차가 발생할 수 있음
- byte와 short은 둘 다 2byte의 크기로 크기는 같지만 서로 범위가 다름
- **둘 중 어느쪽으로의 형변환도 값 손실이 발생할 수 있으므로 자동 형변환이 수행될 수 없음.**

## 정리
1. boolean을 제외한 나머지 7개의 기본형은 서로 형변환 가능
2. 기본형과 참조형은 서로 형변환할 수 없음
3. 서로 다른 타입의 변수간의 연산은 형변환을 하는것이 원칙이지만 값의 범위가 작은 타입에서 큰 타입으로의 형변환은 생략될 수 있음
