## 변수(variable)
### 변수란?
- 수학에서는 변하는 수라고 정의하지만, 프로그래밍 언어에서는 `값을 저장할 수 있는 메모리 상의 공간`을 의미
- 하나의 변수에는 **단 하나의 값만 저장할 수 있으며**, 새로운 값을 저장하면 기존의 값은 사라짐

### 변수의 선언과 초기화
#### 변수의 선언
```declaration
int age; // 정수형 int 타입의 age라는 이름의 변수를 선언
```

- `변수타입` : 변수에 저장될 값이 어떤 `타입(type)`인지를 지정하는 것
- `변수이름` : 변수에 붙인 이름. `메모리 공간에 이름을 붙여주는 것`
- 변수이름을 이용해 저장 공간(변수)에 값을 저장하고, 저장된 값을 읽어올 수 있음
- 서로 구별하기 위해서 같은 이름의 변수가 여러 개 존재해서는 안 됨

#### 변수의 초기화
```initialization
int age = 25; // 변수 age를 선언하고 25로 초기화 한다.
```

- 변수를 사용하기 전에 처음으로 값을 저장하는 것
- **메모리는 여러 프로그램이 공유하는 자원**이므로 전에 다른 프로그램에 의해 저장된 `알 수 없는 값(쓰레기 값, garbage value)`이 남아있을 수 있기 때문에 반드시 **초기화(initialization)** 해야 함
- 변수에 값을 저장할 때는 대입 연산자 `=`를 이용
    - 오른쪽의 값을 왼쪽(변수)에 저장하라는 뜻

```initialization
int a, b;
int x = 0, y = 0;
```

- 변수는 한 줄에 하나씩 선언하는 것이 보통이지만, 타입이 같은 경우 콤마 `,` 를 구분자로 여러 변수를 한 줄에 선언하기도 함
- 변수의 종류에 따라 변수의 초기화를 생략할 수 있는 경우도 있지만 변수는 사용되기 전에 적절한 값으로 초기화 하는 것이 좋음
- **지역변수는 사용되기 전에 반드시 초기화 해야 하지만 클래스변수와 인스턴스변수는 생략할 수 있음**

```VarEx1.java
class VarEx1 {
    public static void main(String[] args) {
        int year = 0;
        int age = 14;

        System.out.println(year);
        System.out.println(age);
        year = age + 2000; // 변수 age의 값에 2000을 더해서 변수 year에 저장
        age = age + 1; // 변수 age에 저장된 값을 1 증가시킨다.

        System.out.println(year);
        System.out.println(age);
    }
}
```

```실행결과
0
14
2014
15
```

#### 두 변수의 값 교환하기
```VarEx2.java
class VarEx2 {
    public static void main(String[] args) {
        int x = 10, y = 20;
        int tmp = 0; // x값을 임시로 저장할 변수를 선언
        
        System.out.println("x:" + x + " y:" + y);

        tmp = x;
        x = y;
        y = tmp;

        System.out.println("x:" + x + " y:" + y);
    }
}
```

```실행결과
x:10 y:20
x:20 y:10
```

### 식별자(identifier)
- 변수의 이름처럼 프로그래밍에서 사용하는 모든 이름
- 같은 영역 내에서 서로 구분(식별)될 수 있어야 한다.

#### 식별자 명명규칙
1. 대소문자가 구분되며 길이에 제한이 없다.
    - `True`와 `true`는 다른 것으로 간주
2. `예약어`를 사용해서는 안 된다.
    - `예약어(Reserved word)` 또는 `키워드(keyword)` : 프로그래밍 언어의 구문에 사용되는 단어
    - 예약어는 **클래스나 변수, 메서드의 이름으로 사용할 수 없음**
    - `true`는 예약어라서 사용할 수 없지만, `True`는 가능하다.
3. 숫자로 시작해서는 안 된다.
    - `top10`은 허용하지만, `7up`은 허용하지 않는다.
4. 특수문자는 `_`와 `$`만을 허용한다.
    - `$harp`은 허용되지만, `S#arp`은 허용되지 않는다.

#### 자바 프로그래머들에게 권장하는 규칙
1. 클래스 이름의 첫 글자는 항상 대문자로 한다.
2. 변수와 메서드의 이름의 첫 글자는 항상 소문자로 한다.
3. 여러 단어로 이루어진 이름은 단어의 첫 글자를 대문자로 한다.
    - `lastIndexOf`, `StringBuffer`
4. 상수의 이름은 모두 대문자로 한다.
    - 여러 단어로 이루어진 경우 `_`로 구분한다.
    - `PI`, `MAX_NUMBER`

## 변수의 타입
### 기본형 변수(primitive type)
- 계산을 위한 `실제 값(data)`을 저장하며 모두 8개
- 자바는 C언어와 달리 참조형 변수 간의 연산을 할 수 없으므로, **실제 연산에 사용되는 것은 모두 기본형 변수**이다.

- 논리형
    - `boolean(1byte)`
    - `true`/`false` 중 하나를 값으로 가진다.
    - 조건식과 논리적 계산에 사용
    - **다른 기본형과의 연산은 불가능**
- 문자형
    - `char(2byte)`
    - 변수에 **하나의 문자**만 저장할 수 있음
    - **문자를 내부적으로 정수(유니코드)로 저장하기 때문에 정수형 또는 실수형과의 연산이 가능**
- 정수형
    - `byte(1byte)`, `short(2byte)`, `int(4byte)`, `long(8byte)`
    - 기본 자료형은 **int**
    - int는 **CPU가 가장 효율적으로 처리할 수 있는 정수형**
    - 효율적인 실행보다 메모리 절약을 위해서는 byte, short을 선택
    - byte는 이진 데이터를 다룰 때 사용
    - short은 C언어와의 호환을 위해 추가
    - 저장할 수 있는 값의 범위는 **-2<sup>n-1</sup> ~ 2<sup>n-1</sup>-1**(n은 bit수)
        - int의 경우 32bit(4byte)이므로 -2<sup>31</sup> ~ 2<sup>31</sup>-1
- 실수형
    - `float(4byte)`, `double(8byte)`
    - 기본 자료형은 **double**
    - float는 정밀도가 7자리
    - double은 정밀도가 15자리
    - 실수형에서는 저장 가능한 값의 범위뿐만 아니라 **정밀도**도 타입 선택의 중요한 기준

### 참조형 변수(reference type)
- 8개의 기본형을 제외한 나머지 타입
- 어떤 값이 저장되어 있는 `메모리 주소(memory address)`를 값으로 가짐
    - 메모리 주소(memory address) : 메모리 위치에 대한 식별자. 1byte 단위로 일련번호가 붙어있다.
- 참조형 변수(참조변수)를 선언할 때는 변수의 타입으로 `클래스 이름`을 사용하므로 **클래스의 이름이 참조변수의 타입**이 됨

```reference_type
클래스이름 변수이름; // 변수의 타입이 기본형이 아닌 것들은 모두 참조변수이다.
```

- 참조변수는 `null` 또는 `객체의 주소(4byte)`를 값으로 가짐
- `null`은 **어떤 객체의 주소도 저장되어 있지 않음**을 뜻함

``` reference_type_initialization
Date today = new Date(); // Date 객체를 생성해서, 그 주소를 today에 저장
```

- 객체를 생성하는 연산자 `new`의 결과는 생성된 객체가 저장된 객체의 주소
- 대입연산자에 의해서 참조변수 today에 저장

## 상수(constant)
- 값을 한 번만 저장할 수 있는 공간
- 한 번 값을 저장하면 다른 값으로 변경할 수 없음
- 프로그래밍 시 변경되지 않아야 되는 값을 상수로 선언함으로써 실수로 변경되지 않도록 방지
- 상수의 이름은 `모두 대문자`로 하는 것이 암묵적인 관례
- 여러 단어로 이루어져 있는 경우 `_`로 구분
- 변수의 타입 앞에 키워드 `final`을 붙여 선언

```constant
final int MAX_SPEED = 10; // 상수 MAX_SPEED를 선언 & 초기화
```

## 리터럴(literal)
- 그 자체로 값을 의미하는 것

```literal
int year = 2014; // year은 변수, 2014는 리터럴
final int MAX_VALUE = 100; // MAX_VALUE는 상수, 100은 리터럴
```

### 리터럴 타입과 접미사
- 논리형
    - true/false
    - 접미사 없음
- 정수형
    - 123, 0b0101, 077, 0xFF, 100L
    - long 타입의 접미사 `l` 또는 `L`을 붙임
        - 숫자 1과 헷갈리기 쉬우므로 `L`을 사용하는 것이 좋음
    - **접미사가 없으면 int 타입의 리터럴**
    - byte와 short 타입의 리터럴은 별도로 존재하지 않음
    - 16진수: 리터럴 앞에 앞에 접두사 `0x` 또는 `0X`
    - 8진수 : 리터럴 앞에 앞에 접두사 `0`
    - 2진수 : 리터럴 앞에 앞에 접두사 `0b`
    - 정수형 리터럴의 중간에 구분자 `_`를 넣어 큰 숫자를 편히 읽을 수 있음
        - long big = 100_000_000_000L;
        - long hex = 0xFFFF_FFFF_FFFF_FFFFL;
- 실수형
    - 3.14, 3.0e8, 1.4f, 0x1.0p-1
    - float 타입의 접미사 `f` 또는 `F`를 붙임
    - double 타입의 접미사 `d` 또는 `D`를 붙임
    - **기본 자료형인 double은 접미사 생략 가능**
    - 소수점이나 10의 제곱을 나타내는 기호 `E` 또는 `e`
    - 2의 제곱을 의미하는 `p`

#### 타입의 불일치
- 타입이 달라도 저장 범위가 넓은 타입에 좁은 타입의 값을 저장하는 것은 허용
- 리터럴의 값이 변수의 타입의 범위를 넘어서면 에러 발생
- 리터럴의 타입이 변수의 타입보다 저장범위가 넓으면 에러 발생

### 문자 리터럴과 문자열 리터럴
- 문자형
    - 'A', '1', '\n'
    - `작은 따옴표`로 문자 하나를 감싼 것
    - 두 문자 이상은 저장하기 위해서는 문자열 사용
    - `''` 안에 **하나의 문자가 반드시 있어야 함**
- 문자열
    - "ABC", "123", "A", "true"
    - `큰 따옴표`로 감싼 것
    - `""` 안에 아무런 문자도 넣지 않는 **빈 문자열 허용**
    - **원래 String은 클래스이지만 객체를 생성하는 연산자 new를 사용하지 않아도 됨**
    - 덧셈 연산자 `+`를 이용하여 문자열을 결합할 수 있음
        - 피연산자가 모두 숫자일 때는 두 숫자를 더하지만, 피연산자 중 어느 한쪽이 String이면 나머지 한 쪽을 먼저 String으로 변환한 다음 두 String을 결합
    - **기본형과 참조형의 구별없이 어떤 타입의 변수도 문자열과 덧셈 연산을 수행하면 그 결과가 문자열이 됨**
    - 기본형 타입 + 빈 문자열("")을 더해주면 문자열이 됨

```StringEx.java
    class StringEx {
    public static void main(String[] args) {
        String name = "Ja" + "va";
        String str = name + "8.0";

        System.out.println(name);
        System.out.println(str);
        System.out.println(7 + " ");
        System.out.println(" " + 7);
        System.out.println(7 + "");
        System.out.println("" + 7);
        System.out.println("" + "");
        System.out.println(7 + 7 + "");
        System.out.println("" + 7 + 7);
    }
}
```


## 형식화된 출력 - printf()
- println()은 사용하기엔 편하지만 변수의 값을 그대로 출력. 값을 변환하지 않고는 다른 형식으로 출력할 수 없음
- printf()는 `지시자(specifier)`를 통해 변수의 값을 여러 가지 형식으로 변환하여 출력할 수 있음
    - 지시자(specifier) : 값을 어떻게 출력할 것인지를 정해주는 역할
- println()과 달리 출력 후 줄바꿈을 하지 않기 때문에 지시자 `\n` 또는 `%n`을 따로 넣어줘야 함

### 자주 사용되는 printf()의 지시자
- %b : 불리언(boolean) 형식으로 출력
- %d : 10진(decimal) 정수의 형식으로 출력
- %o : 8진(octal) 정수의 형식으로 출력
- %x, %X : 16진(hexadecimal) 정수의 형식으로 출력
- %f : 부동 소수점(floating-point)의 형식으로 출력
- %e, %E : 지수(exponent) 표현식의 형식으로 출력
- %c : 문자(character)로 출력
- %s : 문자열(string)로 출력

```PrintfEx1.java
class PrintfEx1 {
    public static void main(String[] args) {
        byte b = 1;
        short s = 2;
        char c = 'A';

        int finger = 10;
        long big = 100_000_000_000L;
        long hex = 0xFFFF_FFFF_FFFF_FFFFL;

        int octNum = 010; // 8진수 10, 10진수로는 8
        int hexNum = 0x10; // 16진수 10, 10진수로는 16
        int binNum = 0b10; // 2진수 10, 10진수로는 2

        System.out.printf("b=%d%n", b);
        System.out.printf("s=%d%n", s);
        System.out.printf("c=%c, %d %n", c, (int)c);
        System.out.printf("finger=[%5d]%n", finger);
        System.out.printf("finger=[%-5d]%n", finger);
        System.out.printf("finger=[%05d]%n", finger);
        System.out.printf("big=%d%n", big);
        System.out.printf("hex=%#x%n", hex); // '#'은 접두사(16진수 0x, 8진수 0)
        System.out.printf("octNum=%o, %d%n", octNum, octNum);
        System.out.printf("hexNum=%x, %d%n", hexNum, hexNum);
        System.out.printf("binNum=%s, %d%n", Integer.toBinaryString(binNum), binNum);
    }
}
```

```실행결과
b=1
s=2
c=A, 65 
finger=[   10]
finger=[10   ]
finger=[00010]
big=100000000000
hex=0xffffffffffffffff
octNum=10, 8
hexNum=10, 16
binNum=10, 2
```

### 정수형 값 출력에 사용되는 지시자
- 지시자 `%d` 사이에 `숫자`를 추가하여 원하는 만큼의 출력 공간 확보

```specifier
System.out.printf("finger=[%5d]%n", finger);
// 5칸만큼 빈 공간을 공백으로 채우며 우측 정렬
// finger=[   10]

System.out.printf("finger=[%-5d]%n", finger);
// 5칸만큼 빈 공간을 공백으로 채우며 좌측 정렬
// finger=[10   ]

System.out.printf("finger=[%05d]%n", finger);
// %05d = 5칸만큼 빈 공간을 0으로 채우며 우측 정렬
// finger=[00010]
```

- 지시자 `%x`와 `%o`에 `#`을 사용하여 접두사 붙이기
    - `%#x`는 `0x`
    - `%#o`는 `0`
    - `%X`에 경우 `접두사`와 `영문자`를 `모두 대문자`로 출력

```specifier
System.out.printf("hex=%x%n", hex);
// hex=ffffffffffffffff

System.out.printf("hex=%#x%n", hex);
// hex=0xffffffffffffffff

System.out.printf("hex=%#X%n", hex);
// HEX=0FFFFFFFFFFFFFFFF
```

```PrintfEx2.java
class PrintfEx2 {
    public static void main(String[] args) {
        String url = "www.codechobo.com";

        float f1 = .10f; // 0.10, 1.0e-1
        float f2 = 1e1f; // 10.0, 1.0e1, 1.0e+1;
        float f3 = 3.14e3f;
        double d = 1.23456789;

        System.out.printf("f1=%f, %e, %g%n", f1, f1, f1);
        System.out.printf("f2=%f, %e, %g%n", f2, f2, f2);
        System.out.printf("f3=%f, %e, %g%n", f3, f3, f3);

        System.out.printf("d=%f%n", d);
        System.out.printf("d=%14.10f%n", d); // 전체 14자리 중 소수점 10자리

        System.out.printf("[12345678901234567890]%n");
        System.out.printf("[%s]%n", url);
        System.out.printf("[%20s]%n", url);
        System.out.printf("[%-20s]%n", url); // 왼쪽 정렬
        System.out.printf("[%.8s]%n", url); // 왼쪽에서 8글자만 출력
    }
}
```

### 실수형 값 출력에 사용되는 지시자
- `%f` : `소수점 아래 6자리`까지만 출력, `소수점 아래 7자리에서 반올림`
- `%e` : `지수 형태`로 출력
- `%g` : `값을 간략하게` 표현

```specifier
System.out.printf("d=%f%n", d);
// d=1.234568
```

- `전체 자리수`와 `소수점 아래의 자리수` 지정
    - `%전체자리.소수점아래자리f`
    - `소수점도 한자리를 차지`
    - `소수점 아래의 빈자리`는 `0`으로 채움
    - `정수의 빈자리`는 `공백`으로 채움

```specifier
System.out.printf("d=%14.10f%n", d);
// 전체 14자리 중 소수점 10자리
// d=  1.2345678900
```
![전체 자리수와 소수점 아래의 자리수 지정](/assets/images/javajungsuk3/2-printf-specifier-decimal.png)

- 지시자 `%s` 사이에 `숫자`를 추가하여 원하는 만큼의 출력 공간 확보
    - 지정된 숫자보다 문자열의 길이가 작으면 빈자리는 `공백`으로 출력
    - `.숫자`를 붙이면 문자열의 일부만 출력

```specifier
System.out.printf("[%s]%n", url);
// 문자열의 길이만큼 출력공간 확보
// [www.codechobo.com]

System.out.printf("[%20s]%n", url);
// 최소 20글자 출력 공간 확보(우측 정렬)
// [   www.codechobo.com]

System.out.printf("[%-20s]%n", url);
// 최소 20글자 출력 공간 확보(좌측 정렬)
// [www.codechobo.com   ]

System.out.printf("[%.8s]%n", url);
// 왼쪽에서 8글자만 출력
[www.code]
```

## 화면에서 입력받기 - Scanner
```ScannerEx.java
import java.util.Scanner;

class ScannerEx {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in); // Scanner 클래스의 객체 생성

        System.out.print("두자리 정수를 하나 입력해주세요. > ");
        String input = scanner.nextLine(); // 입력받은 내용을 input에 저장
        int num = Integer.parseInt(input); // 입력받은 내용을 int 타입의 값으로 변환

        System.out.println("입력내용 : " + input);
        System.out.printf("num = %d%n", num);
    }
}
```
